# This Week in Package Management

Tooling for the weekly roundup post.

## Setup on a new machine

    cd _twipm && bundle install

## Use

    rake feeds              # refresh _data/feeds.json from package-manager feeds
    rake twipm:collect      # gather feeds + extras + bookmarks + git-pkgs + DBLP -> _twipm/collected.json
    rake twipm:opml         # optional: pull latest OPML from ecosyste-ms/package-managers-opml

Then run `/twipm` in Claude Code to draft the post.

## Inputs

- `package-manager.opml` - feed list, regenerated by `rake twipm:opml` from the upstream OPML.
- `extra_feeds.txt` - personal feeds (Mastodon, arXiv search) in newsboat URL-line format. These are twipm-only and don't appear on the public `/feeds` page.
- `bookmarks.txt` - one URL per line, optional note after whitespace. Append things during the week. Cleared after each post.

`collected.json` and `dblp_seen.txt` are gitignored.

## How collection works

`fetch_feeds.rb` (run by `rake feeds`, also hourly in CI) reads `package-manager.opml` and writes `_data/feeds.json` with the last 30 days of items. That file powers the public `/feeds` page and is the feed source for twipm.

`collect.rb` (run by `rake twipm:collect`) reads `_data/feeds.json`, fetches `extra_feeds.txt` via Feedjira, pulls Mastodon boosts from the API, reads `bookmarks.txt`, queries `git-pkgs` and DBLP, and writes everything to `collected.json`.

## Academic papers

arXiv is subscribed as a search feed in `extra_feeds.txt` and arrives via Feedjira like everything else. DBLP has no RSS and only year-level dates, so `collect.rb` queries its search API for a fixed term list, filters to last-year-onwards, and marks anything not already in `dblp_seen.txt` as `new: true` in `collected.json`. First run on a fresh machine reports everything as new.
